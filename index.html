01. 프로젝트 개요

1.1 프로젝트 정의

On-Device Multimodal Driving Copilot은 차량 내부 임베디드 환경에서 카메라 영상, 운전자 텍스트/음성, 차량 매뉴얼, 차량 센서 데이터를 통합 처리하여 실시간 운전 보조 응답을 생성하는 On-Device MLLM 시스템이다.

핵심 기술TensorRT Edge-LLM · Qwen2-VL · ReAct · Reflexion · Plan-and-Execute · Multi-Agent(A2A) · LangGraph · Agentic RAG · Graph RAG(Neo4j) · Text2SQL · MCP · CRAG · LLM-as-a-Judge · AG-UI
프로젝트 기본 정보

기간: 2026.05 ~ 2026.08 (16주)  |  환경: NVIDIA A6000 48GB, Ubuntu x86

다음: 핵심 목표 →

01-2. 핵심 목표

G1On-Device VLM 배포 + 비용 최적화
TensorRT Edge-LLM으로 Qwen2-VL 7B와 1.5B를 각각 INT4(AWQ) 양자화하여 A6000에서 로컬 추론 환경을 구축한다. Plan-and-Execute 패턴을 적용하여 Supervisor Agent(7B)가 계획과 복잡한 판단만 담당하고, 단순한 Tool 호출이나 검색 실행은 1.5B Executor가 처리하는 구조로 평균 토큰 비용을 절감한다.

02. 프로토콜 3층 구조

시스템 통신을 MCP(수직), A2A(수평), AG-UI(프론트) 3개 계층으로 표준화한다.

프로토콜역할적용 범위
MCP (수직)Agent → Tool/Data 연결12종 Vehicle Tool, Qdrant, Neo4j, DB
A2A (수평)Agent → Agent 통신. Agent Card 기반 동적 발견Supervisor ↔ Perception / Knowledge / Execution
AG-UI (프론트)Agent → Frontend 스트리밍WebSocket 토큰, Tool 결과 카드, 대시보드
← 핵심 목표ReAct →

03. ReAct Cognitive Architecture

Perceive → Reason → Act → Observe → Reflect 반복 루프. Reflexion으로 실패 경험을 장기 메모리에 축적.

단계설명구현예시
Perceive입력+카메라+차량 통합 인지Router + VLM + State"비 오는데 창문 열려있어"
ReasonCoT 추론, 계획 수립LLM CoT + 플래닝"비+창문 → 닫기+와이퍼"
Act액션 실행MCP + RAG + SQL + Graphwindow → wiper
Observe성공/실패 판단Result 검증 + State창문 ✓, 와이퍼 실패→재시도
Reflect전략 수정 → 메모리 저장자기평가 + Experience"speed 확인 필요" → 저장
← 프로토콜A2A Agent →

04. A2A Multi-Agent

각 Agent를 A2A Server로 배포. Supervisor가 Agent Card 통해 동적 발견·호출. LangGraph StateGraph 오케스트레이션.

← ReActPlan-Execute →

05. Plan-and-Execute 모델 라우팅

7B Planner가 계획, 1.5B Executor가 실행. 평균 토큰 비용 60~70% 절감.

역할모델담당특성
PlannerQwen2-VL 7B INT4ReAct, 복잡 판단, 계획느리지만 정확
ExecutorQwen2-VL 1.5B INT4Tool, 단순 RAG, SQL빠르고 저렴
VisionQwen2-VL 7B FP16카메라, 환경 감지Vision 시에만
← A2A Agent메모리 →

7.2 Graph RAG 파이프라인

  • 1. Vector RAG: bge-m3 + Qdrant로 관련 문서 추출
  • 2. Entity Extraction: 부품/경고등/증상 엔티티 추출
  • 3. Graph Traversal: Neo4j 1~2 hop 탐색
  • 4. Context Fusion: 벡터+그래프 결과 통합 → LLM 컨텍스트
← 메모리Text2SQL →

08. Text2SQL Agent

8.1 차량 DB

테이블컬럼설명쿼리 예시
vehicle_telemetryts, speed, rpm, fuel, battery, tire_*실시간 센서"어제 평균 속도?"
maintenance_logdate, mileage, type, parts, cost정비 이력"마지막 오일교환?"
dtc_codescode, description, severity, resolvedOBD-II"미해결 경고 목록"
trip_historystart, end, distance, avg_speed, fuel운행 기록"이번 달 총 거리?"

8.2 파이프라인

  • Schema Linking: 관련 테이블만 LLM에 제공
  • Few-shot SQL: 유사 예시를 벡터 검색으로 제공
  • Self-Consistency: N개 SQL → 다수결
  • SQL 검증: 구문+EXPLAIN 후 실행
